## การแบ่งงานหลัก

### **สมาชิกคนที่ 1: ไตรภพ ก๋องใจ**
รับผิดชอบงานหลักดังต่อไปนี้:
* **Backend Development:** พัฒนา `auth-service`, `task-service` และ `log-service`
* **Frontend** พัฒนาหน้าเว็บส่วนติดต่อผู้ใช้ (User Interface)

### **สมาชิกคนที่ 2: นายนัฐพงศ์  รอดอินทร์**
รับผิดชอบงานหลักดังต่อไปนี้:
* **config:**  คอนฟิก Docker Compose
* **System Testing:** ดำเนินการทดสอบ API ผ่าน Postman เพื่อตรวจสอบความถูกต้องตามเกณฑ์ T1-T11
* **Documentation:** จัดเตรียมข้อมูลรายงานสรุปผลการดำเนินงาน และบันทึกภาพหน้าจอ (Screenshots) ของระบบ

## งานที่ดำเนินการร่วมกัน
* ออกแบบ **Architecture Diagram** และวางโครงสร้างการสื่อสารระหว่าง Microservices
* จัดทำไฟล์ **README.md** และตรวจสอบความเรียบร้อยของ Repository ก่อนส่งงาน

## เหตุผลในการแบ่งงาน
แบ่งงานตาความถนัดส่วนบุคคล โดยแยกส่วนโครงสร้างพื้นฐาน (Infrastructure/Backend) ออกจากส่วนการนำเสนอข้อมูล (Frontend) เพื่อให้สามารถพัฒนาไปพร้อมกันได้ 

## สรุปการเชื่อมโยงงานของสมาชิก
งานของสมาชิกทั้งสองคนเชื่อมต่อกันผ่าน **API Gateway (Nginx)** โดยฝั่งพัฒนาจะเตรียมระบบและ Endpoint ไว้ ส่วนฝั่งเอกสารและทดสอบจะทำการยิง Request พร้อมแนบ Token เพื่อตรวจสอบผลลัพธ์ว่าทำงานได้หรือไม่
